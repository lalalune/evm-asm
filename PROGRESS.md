## What "evm-asm is a complete guest program" means

A guest program for L1 stateless block validation must satisfy nine
obligations. evm-asm's current state per obligation:

| # | Obligation | Status |
|---|---|---|
| 1 | RV64 ELF for `riscv64im_zicclsm-unknown-none-elf` | рҹҹЎ substrate вң…; codegen emits `rv64imac` (one extension off) |
| 2 | `read_input` / `write_output` per the IO interface standard | вң… verified specs in `Rv64/SyscallSpecs.lean`; codegen M4 wired |
| 3 | RLP-decode the (block, witness) input | рҹҹЎ pure-Lean RLP вң…; RV64 RLP decoder phases 1вҖ“3 in progress |
| 4 | EVM interpreter loop running on the decoded block | рҹҹЎ `InterpreterLoop.lean` + handler-table simulation theorems вң…; codegen M5 (tiny EVM interp) not yet shipped |
| 5 | Full opcode coverage with verified handlers | рҹҹЎ see axis A.2 below |
| 6 | Accelerator ECALL bridges per `zkvm_accelerators.h` | рҹҹЎ vendored header + per-precompile EL bridges; not yet codegen-wired |
| 7 | MPT verification of pre-state witness proofs | вң— not started |
| 8 | Verified post-state root вҶ’ public output | вң— blocked on 4 + 5 + 6 + 7 |
| 9 | Halt convention per `standard-termination-semantics` | вң… `--halt linux93` default, `docs/host-io-halt-convention.md` |
